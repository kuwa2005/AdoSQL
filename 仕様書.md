# AdoSQL 仕様書

本書は、**コマンドライン対話型の Access データベース用 SQL クライアント**（実行ファイル名 `adosql.exe`）について、**利用者視点および実装要件**として整理した仕様である。**実装言語および配布形態**については §0 を参照する。内部構造の詳細は本書で拘束しない。

---

## 0. 実装方針（確定事項）

| 項目 | 内容 |
|------|------|
| 実装言語 | **ネイティブ**（**C++** を想定）。.NET ランタイム等は不要とする。 |
| 配布・配置 | `adosql.exe` を**任意のディレクトリにコピー**して利用する想定。追加の DLL 同梱は極力避け、単一 EXE を目指す（COM／OS 標準 API のみに依存）。 |
| 設定ファイル | `adosql.ini` は**実行ファイルと同一フォルダ**に置く（§8）。初回起動時に存在しなくてよく、**保存時に当該フォルダへ自動生成**されてよい。 |
| ビットネス | **32bit 版と 64bit 版を別 EXE としてビルド**する。利用する PC にインストールされている **ACE（Access Database Engine）のビット数と一致する EXE** を選ぶこと（不一致時は接続に失敗しうる）。 |
| 起動環境 | **コマンドプロンプト（cmd.exe）** および **PowerShell** からの CLI 利用を想定する。引数・パスのクォートは各シェルの規則に従う。 |
| 入力支援 | 対話中 **Tab キーによる補完**を行う（§3.4）。Windows コンソール上では **行バッファをアプリ側で保持**し、編集キー（Backspace、左右矢印、Home/End、Tab 等）を解釈する方式を推奨する。 |

### 0.1 実装前決定事項（技術方針）

以下は、単一 EXE 配布・保守性・現場での再現性を優先した**実装方針**である。詳細は実装・テストの過程で微調整してよいが、**変更時は本書を改訂**する。

#### 成果物・ビルド・ランタイム

| 項目 | 決定 |
|------|------|
| 配布物の名前 | どちらのビルドも実行ファイル名は **`adosql.exe`** とする。配布は **`adosql-x86.zip`** / **`adosql-x64.zip`**（または同等のフォルダ名）のように**アーカイブ単位でビットネスを区別**し、展開先の `adosql.exe` は常に同名。 |
| ツールチェーン | **Visual Studio 2022**、**C++17 以上**（C++20 可）。プラットフォーム **Win32** / **x64**。 |
| C ランタイム | **静的リンク（`/MT` / `/MTd`）** とし、**VC++ 再配布パッケージ不要**で動作することを優先する。 |
| 対応 OS | **Windows 10 以降（64bit 版 OS 上の x64 EXE）** を正式サポートとする。32bit OS は市場上レアのため**ベストエフォート**（Win32 EXE は 32bit ACE が存在する環境向け）。 |
| バージョン表示 | コマンドライン **`--version` / `-v`**（引数のみで終了、標準出力に製品名・版・コミットやビルド日の目安）を実装する。**`/?` は対話外でも使用法表示**として扱えてよい（§2.1 の使用法表示と内容を揃える）。 |

#### データアクセス層（ADO と補助 API）

| 項目 | 決定 |
|------|------|
| SQL 実行・結果セット | **ADO 2.8 系（`msado15.dll` の `#import`）** を主とする。仕様の「ADO 経由」と一致させ、既存ドキュメント・サンプルとの親和性を取る。 |
| メタデータ（`tablelist` / `querylist` / `describe` / `showsql`） | **ADOX（`msadox.dll`）** および **`OpenSchema`** を**併用**する。Access の保存クエリはプロバイダー実装により **Views / Procedures 等の扱いが変わりうる**ため、**ACE の実挙動に合わせて列挙ロジックを確定**し、ヘルプに「列挙の意味」を短く記す。 |
| 新規 DB 作成（§2.2） | **ADOX `Catalog::Create`** を用い、**`.accdb` を既定の新規形式**とする（ユーザーが `.mdb` パスを指定した場合は、作成可否をエンジン・プロバイダーに委ね、不可ならメッセージで明示）。ロケールは **システム既定ロケール**に合わせ、照合は ACE 既定でよい（日本語環境での実用を優先）。 |

#### 接続文字列・プロバイダー選択

| 項目 | 決定 |
|------|------|
| プロバイダー探索順 | **`.accdb` / `.mdb` とも**、利用可能な **ACE** を優先する。OLE DB プロバイダーは **`Microsoft.ACE.OLEDB.16.0`** を先に試し、失敗時 **`Microsoft.ACE.OLEDB.12.0`** にフォールバックする（環境に無い版はスキップ）。 |
| Jet 単体 | **64bit プロセスでは `Microsoft.Jet.OLEDB.4.0` は使用しない**（制約のため）。32bit ビルドのみ、ACE が無いレガシー環境向けに Jet を**オプションで試す**かどうかは実装段階で判断し、**試す場合は接続エラー時のメッセージを分かりやすく**する。 |
| データベースパスワード | 接続文字列に **`;Jet OLEDB:Database Password=...;`** 形式で付与する（ACE/Jet 系で広く使われる形式）。 |

#### 文字コード・コンソール

| 項目 | 決定 |
|------|------|
| プロセス内のパス・SQL テキスト | **Unicode（UTF-16）** を基準とし、Win32 API は **`wchar_t` / ワイド版**で扱う（`wmain` または同等の引数取得）。 |
| 標準入出力 | 起動時に可能な範囲で **コンソールコードページを UTF-8（65001）** に設定し、**入出力は UTF-8** とする。**レガシー環境で文字化けする場合**は README に「フォント・コードページ」の注意を書く。 |
| `adosql.ini` | **UTF-8（BOM なしで読み書き、エディタ互換のため BOM 付き入力も許容）**。コメント行を推奨。 |
| `@` スクリプト | **UTF-8** とする（BOM 付きを検出した場合は除去して解釈）。改行は **CRLF / LF の両対応**。 |
| Tab 補完（実装方針） | 補完候補の**スキーマ由来の一覧**は、`tablelist` / `querylist` と**同一のフィルタ規則**（§0.1 `tablelist`）に合わせ、必要に応じ**接続確立後にキャッシュ**してよい。候補列挙の件数上限は §9 に従う。 |

#### `@` スクリプトのパス解釈

| 項目 | 決定 |
|------|------|
| 相対パス | **`@` に指定した相対パスは、カレントワーキングディレクトリ（プロセスの CWD）基準**とする。シェルで `cd` した場所と一致するため、現場のバッチ・PowerShell と期待が揃う。 |
| 絶対パス | OS 絶対パスはそのまま用いる。 |

#### `tablelist` の範囲

| 項目 | 決定 |
|------|------|
| 既定の列挙対象 | **ユーザー定義およびリンクテーブル等、ユーザーが参照する対象として意味のあるテーブル**を含める。 |
| 除外 | **`MSys*` など Access のシステムテーブル**は**既定では列挙しない**（誤操作防止・ノイズ低減）。将来、拡張で「システムテーブル表示」オプションを足す余地は残す。 |

#### 終了コード

| コード | 意味 |
|--------|------|
| **0** | 正常終了。使用法の表示のみ、対話の `exit` / `quit`、接続後の通常終了を含む。 |
| **1** | 異常終了。起動時の引数不正、ACE 不在・接続失敗、DB ファイルオープン不可など**対話ループに入れない**致命的エラー。 |
| **対話中の SQL エラー** | **当該コマンドは失敗メッセージのみ**とし、**プロセスは継続**（終了コードはセッション終了時まで原則変更しない）。必要になった場合のみ将来、設定で「エラー時即終了」を検討。 |

#### パスワード入力の拡張（任意機能の扱い）

| 項目 | 決定 |
|------|------|
| 第1版 | 仕様どおり **コマンドライン第2引数のみ**とする。 |
| 将来 | **環境変数**（例: `ADOSQL_DB_PASSWORD`）や**対話プロンプト入力**は、セキュリティと実装コストを踏まえ**後続版でオプション追加**可能とする。 |

## 1. 製品の位置づけ

| 項目 | 内容 |
|------|------|
| 名称 | AdoSQL（仮称。実行ファイル `adosql.exe`） |
| 目的 | **Microsoft Access が扱うデータベース**（`.accdb` を主対象、環境に応じ `.mdb` も）に対し、**ADO（ActiveX Data Objects）** 経由で SQL を対話実行する。 |
| 実行形態 | Windows 上のコンソール（ターミナル）から利用する。 |
| SQL 方言 | 接続先エンジン（ACE / Jet）が解釈する Access SQL に従う。 |

前提として、対象 PC に **Microsoft Access Database Engine（ACE）** 等、ADO から利用可能な **OLE DB プロバイダー** が存在することを求める。未インストール時は、分かりやすいエラーメッセージで終了する。

## 2. 起動・接続

### 2.1 コマンドライン引数

```text
adosql.exe <データベースファイルのパス> [パスワード]
```

- **第1引数（必須）**: データベースファイルへのパス。フルパスへの正規化を試みる。
- **第2引数（任意）**: データベースパスワード。指定時は接続文字列に **`Jet OLEDB:Database Password`** 形式で反映する（§0.1）。
- **引数不足時**: 簡潔な使用法を表示し、終了コード 0 で終了する。

空白を含むパスは、OS のシェル規則に従いクォートで囲む。

### 2.2 ファイル未存在時

指定パスにファイルが存在しない場合、**新規データベースを作成するか**をユーザーに確認する（`y` / `n` 等）。作成する場合は、日本語環境で利用しやすい既定ロケール／照合順序を選ぶことを推奨する（具体値は実装判断）。

### 2.3 接続失敗時

ファイル破損、パスワード誤り、プロバイダー非対応、ロック競合など、接続確立に失敗した場合はエラー内容を表示して終了、または対話ループに入らない。

## 3. 対話モード（入力モデル）

### 3.1 プロンプト

- 1 行目のプロンプトは `SQL>` とする。
- 複数行入力中は、2 行目以降を `2>`, `3>`, … のように行番号付きで表示する。

### 3.2 複数行 SQL

- 入力行を蓄積し、**セミコロン `;` が出現した時点**で 1 文として確定・実行する。
- **空行のみの入力**（改行のみ）は、入力の打ち切り（キャンセル）とし、蓄積を破棄してプロンプトを `SQL>` に戻す。

### 3.3 標準入力の行長

- 1 行あたりの読み取り上限は **2048 文字**（終端のヌル除く）を上限とする（§9 の上限と一致させる）。
- 超過分は当該行の残りを破棄し、`Input line too long` に相当するメッセージを出して、**その入力のやり直し**とする。

### 3.4 Tab キーによる補完

対話モードで**現在編集中の 1 行**において **Tab** を押したとき、**カーソル位置**を基準に補完を行う。

#### 3.4.1 振る舞い（共通）

- **補完対象**は、カーソルが**トークン内部または直後**にあるとき、その**トークン全体**（識別子・パス片・キーワード片）とする。空白のみの行では補完しない。
- **候補が 1 つ**のとき: 当該トークンを**候補で置換**し、必要なら**末尾に区切り文字**（空白や `(` 等、文脈に応じた最小限）を挿入してもよい。
- **候補が複数**のとき:
  - 初回 Tab: **全候補に共通するプレフィックス**までを挿入・拡張する（readline 互換の挙動）。
  - **同一トークンに対する続きの Tab**: **候補一覧**を標準出力に表示する（アルファベット順など一貫した順）。表示後、**プロンプトと現在の入力行を再描画**し、編集を継続できること。
- **候補が 0** のとき: **入力文字列は変更しない**。ベル音（コンソールベル）を鳴らしても、無音のままでもよい。
- **Shift+Tab**（逆方向の候補切替）は**必須としない**（実装してもよい）。

#### 3.4.2 補完の種類と優先

以下の**複数種類の候補**が重なる場合は、**マージした一覧**からフィルタする。表示順は実装で定め、**ヘルプに概要**を記す。

| 種類 | 条件（例） | 候補の内容 |
|------|------------|------------|
| **対話コマンド** | 行先頭、または `;` 直後など**新しい文の先頭**に相当する位置で、入力中のトークンがコマンド語のプレフィックスである | §4 の先頭キーワード（`exit`, `select`, `set`, `describe`, `tablelist` 等）、および **`set` の第 2 語**（`pagesize`, `linesize`, `maxrows`, `widthclamp`）、**`widthclamp` の `on` / `off`** 等、実装が認識する固定語 |
| **`@` スクリプトパス** | 行が `@` で始まり、カーソルが `@` 以降のパス部分にある | **ファイルシステム**上のファイル・ディレクトリ名。基準ディレクトリは **カレントワーキングディレクトリ**（§0.1）。ディレクトリには末尾 `\` を付けて次の階層入力に進めることを推奨 |
| **スキーマ（テーブル・クエリ）** | **接続確立後**かつ、**`describe` / `desc` / `showsql` の直後のトークン**として解釈できる位置、または **SQL 文として識別子として妥当な位置**（実装は**単純なトークン境界**に基づき、誤補完を減らすヒューリスティックを適用してよい） | **`tablelist` / `querylist` と同一規則**で列挙される**テーブル名・保存クエリ名**（§0.1） |
| **SQL キーワード** | 上記以外で、識別子トークンのプレフィックスとして補完する場合 | Access SQL で頻出する語の**固定リスト**（`SELECT`, `FROM`, `WHERE`, `INSERT`, `UPDATE`, `DELETE`, `JOIN`, `INNER`, `LEFT`, `ON`, `GROUP`, `ORDER`, `BY`, `AS`, …）。**大文字小文字**は補完後**小文字寄せ**でも**入力のまま**でもよい（エンジン解釈に委ねる） |

未接続時は**スキーマ補完**は行わない（候補に含めない）。

#### 3.4.3 ドキュメント

- `help` の出力に、**Tab で補完できること**を**簡潔に**含める。

## 4. 先頭キーワードによるコマンド分類

**先頭トークン**のみ大文字小文字を区別せずに解釈する（SQL 本体の識別子・リテラルはエンジンに委ねる）。

| 分類 | 入力の先頭 | 動作 |
|------|------------|------|
| 終了 | `exit`, `quit` | プロセス終了 |
| ヘルプ | `help`, `?`, `/?` | 利用可能な対話コマンド一覧を表示 |
| コメント | `//` で始まる行 | 何も実行しない |
| スクリプト | `@<ファイルパス>` | 指定ファイルを開き、行を順に対話ループへ供給する（ネスト可）。**信頼できるローカルファイルのみ**を想定し、パストラバーサル等に注意する実装とする。**相対パスはカレントワーキングディレクトリ基準**（§0.1）。 |
| 表示設定 | `set pagesize <n>` | `SELECT` 結果の 1 ページあたりの**表示行数**（画面行の目安）。許容範囲 **5〜500**。 |
| 表示設定 | `set linesize <n>` | 表示の**横幅**（文字数目安）。許容範囲 **20〜500**。 |
| 表示設定 | `set maxrows <n>` | 1 ページで読み込み表示する**最大レコード数**。`0` で無制限（または無効）。正の整数でページ分割の閾値とする。 |
| 表示設定 | `set widthclamp on` / `off` | `on` 時、`linesize` を超える列は表示から外し、表の横幅を抑える。 |
| 表示設定 | `show settings` | 現在の `pagesize`, `linesize`, `maxrows`, `widthclamp` 等を表示。 |
| ユーティリティ | `prompt <任意テキスト>` | メッセージを標準出力に表示（スクリプト向け）。 |
| DML | `select` | 結果セットを表形式で表示。 |
| DML | `insert`, `update`, `delete` | 下記「トランザクション」と組み合わせて実行。 |
| DDL 等 | `create`, `alter`, `drop` | エンジンが受け付ける文を実行。 |
| トランザクション | `commit`, `rollback` | 文のみ、または文＋セミコロンのみでも受け付ける。 |
| メタ | `describe` / `desc` | 続く識別子をテーブル名または保存クエリ名とみなし、列定義を表示。 |
| メタ | `showsql` | 保存クエリ（クエリ定義）の SQL 本文を表示する。 |
| メタ | `tablelist` | テーブル一覧。**既定では `MSys*` 等のシステムテーブルを除く**（§0.1）。 |
| メタ | `querylist` | クエリ（クエリ定義）一覧を表示。 |

上記のいずれにも合致しない先頭トークンは `Invalid Command` に相当するメッセージを表示する。

---

## 5. トランザクション（DML）

- `insert` / `update` / `delete` 実行前に、未開始であればトランザクションを開始する。
- **`commit` まで永続化しない**動作とする（明示的コミット前はロールバック可能）。
- 対話セッション終了時、未コミットの変更があれば **`rollback`** 相当を行い、暗黙のコミットをしない。

---

## 6. `SELECT` 結果の表示

- 読み取り専用の結果セットとしてオープンする。
- フィールド型に応じた整形（数値の右寄せ、日付・時刻の書式、NULL の表現など）を行う。
- **既定値の目安**: `linesize = 79`, `pagesize = 24`（`set` で変更可能）。
- **`maxrows > 0` のとき**: 指定件数ごとに `--More--` を表示し、**Enter** で続行、**`q`** で打ち切り。
- **`widthclamp = on` のとき**: `linesize` を超える列は表示対象から除外し、横幅を抑える。
- テキスト型フィールド内の改行は、画面上では `<crlf>`, `<lf>`, `<cr>` 等の可視プレースホルダに置換して表示する。

式のみの列・集約列は、エンジンが付与する列名が分かりにくい場合があるため、**`AS` による別名**の利用を推奨する旨をヘルプまたはドキュメントに記す。

---

## 7. エラー表示

- データアクセス例外が発生した場合、**ユーザー向けに安全な範囲**でメッセージを表示する。
- プロバイダーが詳細エラー情報を返さない場合でも、**空参照にならない**ようフォールバック文言を用意する。

---

## 8. 設定の永続化

- `pagesize`, `linesize`, `maxrows`, `widthclamp` 等の表示設定は、**実行ファイルと同一フォルダ**に置く設定ファイル（例: `adosql.ini`）に保存する。
- 設定ファイルが存在しない場合は既定値で起動し、**初回保存時にファイルを生成**してよい。
- 設定ファイルの形式（INI 等）は実装が選ぶ。コメント付きで人間が編集しやすいことを推奨する。**文字コードは UTF-8**（§0.1）。

---

## 9. 上限・制約（仕様レベル）

| 項目 | 値・内容 |
|------|-----------|
| 1 行入力 | 最大 2048 文字（超過時は破棄・再入力） |
| メタデータ反復 | テーブル・クエリ・フィールド等の列挙は、**最大 255 件**を上限とし、超過時はエラー表示のうえ処理を中断してよい |
| Tab 補完の一覧表示 | 1 回の Tab 連打で標準出力に列挙する候補は**最大 255 件**とし、超過分は省略し、**省略した旨**を表示してよい（メタデータ反復の上限と整合） |
| 同時接続 | 単一プロセス・単一接続を基本とする（複数接続は本書の対象外） |
