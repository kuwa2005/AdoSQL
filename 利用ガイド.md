# AdoSQL 利用ガイド

AdoSQL は、Windows のコマンドラインで Access ファイルに SQL を実行するためのツールです。

## 1. 事前準備

- Windows 環境
- Access ファイル（`.accdb` または `.mdb`）
- Microsoft Access Database Engine (ACE) がインストール済みであること
  - 64bit 版 `adosql.exe` には 64bit ACE
  - 32bit 版 `adosql.exe` には 32bit ACE

## 2. 起動方法

PowerShell では `.\` を付けて起動します。

```powershell
.\adosql.exe "C:\data\sample.accdb"
```

パスワード付き DB の場合:

```powershell
.\adosql.exe "C:\data\sample.accdb" your_password
```

> 注意: パスワードを引数で渡すと、履歴やプロセス一覧に残ることがあります。

## 3. よく使うコマンド

- `help` : ヘルプ表示
- `tablelist` : テーブル一覧
- `querylist` : 保存クエリ一覧
- `desc テーブル名` : 列情報表示（`describe` でも可）
- `showsql クエリ名` : 保存クエリの SQL 表示
- `set pagesize 40` : 1ページ表示行数
- `set linesize 120` : 表示横幅
- `set maxrows 100` : `--More--` の区切り件数（`0` で無効）
- `set widthclamp on` : 横幅超過列を非表示にして崩れを防ぐ
- `show settings` : 現在設定表示
- `cls` / `clear` : 画面クリア
- `commit` / `rollback` : トランザクション制御
- `exit` / `quit` : 終了

## 4. SQL 実行の基本

- SQL は `;` で確定して実行されます。
- `help` や `tablelist` などのシステムコマンドは `;` なしで実行できます。

例:

```sql
select * from 商品マスタ;
update 商品マスタ set 価格 = 120 where 商品ID = 1;
commit
```

## 5. スクリプト実行

`@` を付けるとファイルの内容を順に実行します（UTF-8 推奨）。

```text
@scripts\init.sql
```

## 6. 表示のコツ

- 横表示が崩れる場合: `set linesize 100` などで調整
- さらに崩れる場合: `set widthclamp on`
- 長い結果は `set maxrows 100` で分割表示にすると見やすい

## 7. トラブル時

- 「接続できない」「Provider がない」:
  - ACE が入っているか確認
  - EXE のビット数と ACE のビット数が一致しているか確認
- PowerShell で起動できない:
  - `adosql` ではなく `.\adosql.exe` で実行

---

困ったときは `help` を最初に実行してください。
